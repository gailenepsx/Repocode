The current simulation process supports the business review of AML scenarios by transforming input transaction data into usable alert outputs for analysis. The process starts when the simulator receives the Payments and Postings CSV files and prepares the data required to run the simulation. It then uses the parameter file to execute the configured models, identify potential hits, and convert those hits into alerts. These alerts are subsequently organized and consolidated according to the business logic of each model so that they can be scored and reviewed consistently. At the end of the process, the simulator produces the alert output files by model, a consolidated output for all models, the associated alert notes and alert steps files, as well as audit trail outputs that support result traceability and threshold analysis.
